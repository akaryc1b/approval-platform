# 功能对齐矩阵

本矩阵用于追踪钉钉、飞书审批领域公开可验证能力。它描述功能等价目标，不代表复制第三方实现或视觉。

状态：`Planned`、`Designing`、`In progress`、`Available`、`Verified`。

| 能力域 | 功能 | 目标 | 状态 |
|---|---|---|---|
| 流程设计 | 树状审批设计器 | 普通管理员无需理解 BPMN | Planned |
| 流程设计 | 条件与并行分支 | 支持嵌套、优先级和汇聚校验 | Planned |
| 流程设计 | 子流程、定时、自动化、Webhook | 支持同步与异步节点 | Planned |
| 审批模式 | 单人、或签、并行会签 | 完整支持 | Planned |
| 审批模式 | 顺序会签、固定票数、比例、权重 | 完整支持 | Planned |
| 审批人 | 用户、角色、岗位、部门负责人 | 快照与实时解析策略 | Planned |
| 审批人 | 多级主管、自选、表单字段、动态接口 | 支持空审批人和去重策略 | Planned |
| 表单 | 基础字段、明细、附件、人员部门 | PC 与移动端同 Schema | Planned |
| 表单 | 联动、公式、数据引用、节点权限 | 版本化与可审计 | Planned |
| 表单 | 签名、签章、OCR、打印、多语言 | Provider 化 | Planned |
| 任务 | 同意、拒绝、任意驳回 | 并行、多实例组合正确 | Planned |
| 任务 | 撤回、拿回、转办、委派、协办 | 严格操作语义 | Planned |
| 任务 | 前加签、后加签、减签 | 支持顺序与并行 | Planned |
| 代理 | 定时代理、流程范围、冲突策略 | 防循环代理 | Planned |
| 代理 | 离职转办、流程交接 | 批量预演和审计 | Planned |
| 协作 | 抄送、已阅、催办、评论、@、附件 | 多渠道通知 | Planned |
| 时间 | 工作日历、SLA、超时升级 | 分布式幂等执行 | Planned |
| 工作台 | 待办、已办、我发起、抄送、草稿 | PC 与移动端 | Planned |
| 工作台 | 搜索、筛选、关注、批量审批 | 权限和风险限制 | Planned |
| 管理 | 版本、灰度、复制、导入导出 | 运行实例不可变 | Planned |
| 管理 | 跨组织审批、第三方待办 | Connector 实现 | Planned |
| 运维 | Job、卡死流程、无审批人、回调失败 | 统一告警与诊断 | Planned |
| 运维 | 跳转、重启、迁移、补偿、消息补发 | 预演、双人复核、审计 | Planned |
| 分析 | 耗时、瓶颈、超时率、人员负载 | 租户隔离 | Planned |
| AI | 摘要、材料检查、风险识别、意见生成 | 默认人工确认 | Planned |
| AI | 路径和审批人推荐、自动审批 | 白名单和强审计 | Planned |

每个 `Verified` 项必须具备产品验收用例、自动化测试和对应文档。
