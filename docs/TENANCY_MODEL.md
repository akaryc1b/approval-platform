# 多租户模型

## 原则

- 每张平台业务表都包含不可为空的 `tenant_id`；
- 缓存、对象存储、搜索索引、消息和 Job 均携带租户上下文；
- Flowable 部署、定义、实例和任务必须设置租户标识；
- 不允许从用户输入直接构造忽略租户的查询；
- 平台管理员跨租户访问必须显式切换、二次授权并审计。

## 数据隔离

初期采用共享数据库、共享 Schema、行级租户隔离。设计时保留独立 Schema/数据库部署能力，但业务代码只依赖 TenantContext 和 Repository 端口。

## 异步处理

Outbox、定时任务、消息消费和失败重试都必须从持久化事件恢复租户上下文，禁止依赖线程本地残留状态。

## 宿主映射

同一个外部组织可以映射到一个平台租户。外部用户 ID 由 `connectorType + connectorInstance + externalId` 唯一确定，避免不同系统 ID 冲突。
