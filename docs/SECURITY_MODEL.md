# 安全模型

## 信任边界

平台区分最终用户、宿主系统、连接器、平台管理员、租户管理员和运维管理员。任何请求不得仅凭前端菜单判断权限。

## 身份与授权

- 用户请求使用 OIDC/JWT 或连接器换取的平台会话；
- 系统调用使用独立客户端凭据和细粒度 Scope；
- 审批操作同时校验任务参与关系、租户、数据权限和操作状态；
- 高风险运维操作支持双人复核和临时授权。

## 数据保护

- 敏感字段分类、脱敏和可选列级加密；
- 附件使用短期签名 URL；
- AI Provider 只接收授权字段，外部模型调用前执行脱敏；
- 日志禁止记录完整 Token、密码、签名密钥和敏感表单正文。

## 防护

- 幂等键与乐观锁防止重复审批；
- Webhook 签名、时间窗口和 nonce 防重放；
- 上传文件执行类型、大小、病毒和内容安全检查；
- 表达式、脚本和自动化节点采用受限执行环境；
- 管理接口使用独立权限域和审计通道。
